# ETL Pipeline — Amman Digital Market

## Overview

This project implements a complete ETL (Extract, Transform, Load) pipeline using Python.

The pipeline:

- Extracts data from a PostgreSQL database (customers, products, orders, order_items)
- Transforms the data into customer-level analytics using Pandas
- Validates data quality with multiple checks
- Loads the final dataset into a new PostgreSQL table and a CSV file

The goal is to generate a clean customer analytics dataset for business insights.

---

## Setup

1. Start PostgreSQL container:
   ```bash
   docker run -d --name postgres-m3-int \
     -e POSTGRES_USER=postgres -e POSTGRES_PASSWORD=postgres \
     -e POSTGRES_DB=amman_market \
     -p 5432:5432 -v pgdata_m3_int:/var/lib/postgresql/data \
     postgres:15-alpine
   ```
2. Load schema and data:
   ```bash
   psql -h localhost -U postgres -d amman_market -f schema.sql
   psql -h localhost -U postgres -d amman_market -f seed_data.sql
   ```
3. Install dependencies: `pip install -r requirements.txt`

## How to Run

```bash
python etl_pipeline.py
```

## Output

The pipeline generates a customer-level analytics dataset saved as:

- `output/customer_analytics.csv`
- PostgreSQL table: `customer_analytics`

### The dataset contains the following columns:

- `customer_id` — unique identifier for each customer
- `customer_name` — customer name
- `city` — customer city (may contain null values)
- `total_orders` — number of distinct valid orders per customer
- `total_revenue` — total revenue generated by the customer
- `avg_order_value` — average revenue per order (total_revenue / total_orders)
- `top_category` — the product category that generated the highest revenue for the customer

This dataset provides a clean summary of customer behavior and spending.

---

## Quality Checks

The pipeline performs several data validation checks to ensure accuracy and reliability:

- **No null customer_id**  
  Ensures every record has a valid unique identifier

- **No null customer_name**  
  Ensures customer names are present and usable

- **Positive total_revenue**  
  Ensures only meaningful financial data is included

- **No duplicate customer_id**  
  Guarantees one record per customer

- **Positive total_orders**  
  Ensures each customer has at least one valid order

If any of these checks fail, the pipeline raises a `ValueError` and stops execution to prevent loading invalid data.
